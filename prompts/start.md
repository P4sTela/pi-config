# Start a coding task

Before changing files, perform a concise repository preflight for this task: $@

- Identify the relevant entry points, existing patterns, and nearby tests.
- Check the current git status and avoid overwriting unrelated user changes.
- State the smallest reasonable implementation plan and any blocking ambiguity.
- If the task is clear, proceed with implementation after the preflight; do not stop at a plan.
- Keep the scope focused and validate the changed behavior before finishing.
